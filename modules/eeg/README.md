# PRĀNA EEG Module (Phase 2)

Standalone EEG classification module for the PRĀNA AI-Powered Multi-Modal Patient Triage System. This module is **not yet integrated** with NATS, the dashboard, or the fusion engine.

## Directory layout

```
modules/eeg/
├── dataset/              # Place EEG data here (auto-downloads UCI if empty)
│   ├── data.csv          # UCI Epileptic Seizure Recognition (default)
│   ├── dataset_meta.json # Generated by inspect_dataset.py
│   └── example_signal.png
├── model/
│   ├── eeg_model.pth     # Best trained CNN checkpoint
│   └── training_history.png
├── utils.py              # Paths, loading, format detection
├── inspect_dataset.py    # Step 1 — dataset inspection
├── preprocessing.py      # Step 2 — filtering, normalization, splits
├── eeg_dataset.py        # Step 3 — PyTorch Dataset / DataLoader
├── eeg_model.py          # Step 4 — 1D CNN architecture
├── train_eeg.py          # Step 5 — training with early stopping
├── evaluate_model.py     # Step 6 — test-set metrics
├── eeg_inference.py      # Step 7 — single-sample prediction
└── README.md
```

## Quick start

Run from the **project root**:

```bash
# 1. Inspect dataset (downloads UCI CSV if dataset/ is empty)
python modules/eeg/inspect_dataset.py

# 2. Train the 1D CNN
python modules/eeg/train_eeg.py

# 3. Evaluate on held-out test set
python modules/eeg/evaluate_model.py

# 4. Run inference demo
python modules/eeg/eeg_inference.py
```

## Dataset

### Default: UCI Epileptic Seizure Recognition

If `dataset/` is empty, `inspect_dataset.py` automatically downloads:

- **Source:** [UCI ML Repository — Epileptic Seizure Recognition](https://archive.ics.uci.edu/ml/datasets/Epileptic+Seizure+Recognition)
- **Samples:** 11,500 EEG windows
- **Signal length:** 178 time points per window
- **Sampling rate:** 173.61 Hz (downsampled from 4097 points)
- **Classes:** 5 (class 1 = seizure, classes 2–5 = non-seizure)

### Clinical binary mapping

| Raw class | Clinical status |
|-----------|-----------------|
| 1         | ABNORMAL (seizure) |
| 2 – 5     | NORMAL (non-seizure) |

### Custom datasets

Place files in `modules/eeg/dataset/`:

| Format | Expected layout |
|--------|-----------------|
| `.csv` | Numeric feature columns + label column (`label`, `y`, `class`, or last column) |
| `.npz` | Keys: `signals`/`X` + `labels`/`y` |
| `.mat` | Arrays with signal/label naming hints |
| `.npy` | `(N, L)` array of windows |

Re-run `inspect_dataset.py` after adding data so preprocessing adapts automatically.

## Preprocessing

`preprocessing.py` provides reusable functions:

1. **Band-pass filter** — Butterworth 0.5–45 Hz (when sampling rate is known)
2. **Normalization** — Z-score (default) or min-max
3. **Window segmentation** — For long continuous recordings
4. **Label encoding** — Binary NORMAL (0) / ABNORMAL (1)
5. **Train / val / test split** — Stratified 70 / 10 / 20

Configuration is derived from `dataset/dataset_meta.json` produced during inspection.

## CNN architecture

`EEG1DCNN` in `eeg_model.py`:

```
Input (B, 1, L)
  → Conv1D(1→32) → BatchNorm → ReLU → MaxPool
  → Conv1D(32→64) → BatchNorm → ReLU → MaxPool
  → Conv1D(64→128) → BatchNorm → ReLU → MaxPool
  → Flatten
  → Linear → ReLU → Dropout(0.5)
  → Linear → Output (num_classes)
```

Hyper-parameters (`conv_channels`, `kernel_size`, `fc_hidden`, `dropout`) are configurable via `build_model()`.

## Training

`train_eeg.py`:

- **Loss:** CrossEntropyLoss
- **Optimizer:** Adam (lr=1e-3)
- **Scheduler:** ReduceLROnPlateau
- **Early stopping:** patience=7 on validation loss
- **Output:** `model/eeg_model.pth` + `model/training_history.png`

## Evaluation

`evaluate_model.py` reports on the held-out test set:

- Accuracy, Precision, Recall, F1-score, ROC AUC
- Confusion matrix (console + `model/confusion_matrix.png`)
- ROC curve (`model/roc_curve.png`)
- Classification report
